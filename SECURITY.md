# Security Policy

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use the repository's
private **Security > Advisories > Report a vulnerability** flow and include the
affected URL or component, reproduction steps, impact, and any suggested fix.

Do not access another tenant's data, disrupt production, or retain candidate
data while testing. We will acknowledge a complete report within two business
days, provide a severity assessment within five business days, and coordinate
remediation and disclosure with the reporter.

## Supported version

Security fixes are applied to the current production branch. Production must be
deployed from a reviewed commit with passing security and regression checks.
